# 한국어 발표 문장

한국어 슬라이드와 발표자 노트를 만들 때 읽는다. 아래 원칙은 [epoko77-ai/im-not-ai](https://github.com/epoko77-ai/im-not-ai)의 공개 규칙에서 발표 자료에 맞는 항목을 추려 적용한 것이다.

## 이름과 언어

- 공식 한국어 이름이 확인되면 한국어를 기본 표기로 쓴다. 원어는 첫 등장에만 작게 병기한다.
- 공식 한국어 이름이 없으면 고유명사를 멋대로 번역하지 않는다. 자연스러운 음역 또는 원어를 쓰고, 청중이 이해해야 하는 설명만 한국어로 덧붙인다.
- 영어 키커와 영어 카드 제목은 장식이 아니라 정보일 때만 쓴다. 한국어로 충분하면 한국어를 쓴다.
- 업계 표준 약어, 규격, 모델명, 수치, 날짜는 보존한다.
- 생소한 약어·기관·시장·제도는 청중이 현재 슬라이드를 이해하는 데 꼭 필요할 때만 첫 등장에 짧게 설명한다. 도메인 실무자가 모두 아는 용어에는 달지 않는다.
- 용어 주석은 `용어 — 이 발표에서 뜻하는 역할`처럼 한 줄로 쓴다. 약어의 정식 명칭을 확인하지 못했다면 억지로 풀어 쓰지 않는다.

## 문장

- 발표자가 입으로 말할 법한 짧고 구체적인 문장을 쓴다.
- “~를 통해”, “~에 있어서”, “~에 기반하여”, “가지고 있다”를 습관처럼 쓰지 않는다. 조사나 동사로 바로 연결한다.
- “결론적으로”, “주목할 만하다”, “시사하는 바가 크다”, “본질적으로”, “지금이야말로” 같은 상투어를 피한다.
- “첫째·둘째·셋째”식 기계적 병렬과 똑같은 세 카드 구성을 반복하지 않는다.
- “또한·따라서·즉·나아가”로 문장을 연달아 시작하지 않는다.
- 명사형 표현을 동사로 풀고, 피동보다 능동을 우선한다.
- 같은 길이와 같은 종결어미가 네 문장 이상 이어지지 않게 리듬을 섞는다.
- 과장 형용사 대신 장면, 행동, 숫자, 조건을 보여준다.
- 원문에 없던 비유나 문학적 수사를 억지로 만들지 않는다.

## 슬라이드 점검

1. 한국어로 바꿔도 정보가 줄지 않는 영어 표현이 남았는가?
2. 제품명·게임명·수치·날짜·직접 인용이 출처와 일치하는가?
3. 제목이 광고 문구처럼 과장되거나 번역투로 굳어 있지 않은가?
4. 본문이 발표자가 한 호흡에 읽을 수 있는가?
5. 발표자 노트가 슬라이드 문구를 그대로 반복하지 않는가?
6. 청중에게 생소하면서 이해에 꼭 필요한 용어만 설명했고, 익숙하거나 중요하지 않은 용어까지 주석으로 늘어놓지 않았는가?
